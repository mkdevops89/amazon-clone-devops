# 📦 Amazon-Like E-Commerce Platform (Phase 8.5: Advanced Observability)

## 🚀 Phase 8.5 Overview
This branch (`phase-8.5-advancedobservability`) represents the capstone to our Day-2 Operations journey by introducing **"Eyes on Glass" Advanced Observability**. 

Previous phases built self-healing automations, hybrid Spot Node architecture, and intelligent Kubernetes scheduling. In this phase, we add the visual layer necessary for engineers and FinOps teams to actively monitor these systems.

By deploying custom Grafana Dashboards and integrating specialized Prometheus ServiceMonitors, we gain real-time visibility into the "Three Pillars" of our infrastructure: the financial savings generated by our Spot architecture, the deep hardware health of our cluster nodes, and the golden signals of our application traffic.

### 📊 Observability Architecture ("The Three Pillars")
1. **Pillar 1: FinOps & Efficiency Dashboard (`finops.json`)**
   * **Purpose**: Visualizes the direct financial impact of the Phase 8 Hybrid Architecture.
   * **Metrics**: Compares the count of running Spot Instances vs On-Demand capacity, and calculates the estimated monthly AWS bill based on active spot discounts.
2. **Pillar 2: Node Health Dashboard (`nodes.json`)**
   * **Purpose**: Visualizes the stability of the EKS cluster to validate the Phase 7 Right-Sizing and Scaling AI.
   * **Metrics**: Actively tracks Memory Pressure and Disk Pressure alerts natively from the `kubelet`.
3. **Pillar 3: App Golden Signals (`apps.json`)**
   * **Purpose**: Visualizes the real-time user experience of the Amazon-like Backend.
   * **Metrics**: Monitors Latency, Error Rates (HTTP 5xx), and overall Request Throughput (enabled via a custom Prometheus `ServiceMonitor`).
